# トラブルシューティング

症状別に「どこを見るか」「何を試すか」をまとめます。  
まずは **Consoleの赤エラー** を優先して潰すのが近道です。

---

## まず確認すること（共通）

1. VCC で おちびちゃんズ化ツール（Ochibi-chans Converter Tool） が **Installed** になっている
2. Unity を再起動（初回インストール直後は読み込みが入ります）
3. Unity Console に赤エラーが出ていない
4. **HierarchyでScene上のアバター**を選んでいる（Project上のPrefabではない）

---

## ダイアログが出る（文言そのまま）

### 「Hierarchy で元のアバターを 1 つ選んでください。」

- Hierarchyで **アバターのルート**を1つ選んでから、もう一度実行してください。
- Project ビューで Prefab を選んでいると対象になりません。

### 「Project でおちびちゃんズの Prefab を 1 つ選んでください。」

- 変換先のおちびちゃんズ Prefab を **手動指定**してください。  
  Project から Prefab をツールウィンドウにドラッグ＆ドロップします。

---

## 候補（プルダウン）が出ない / 間違った候補になる

自動候補は、顔メッシュ（Viseme用メッシュ）を手がかりに **`Assets/夕時茶屋`** 配下を走査します（ツール定数 `BaseFolder`）。

### まず試す

1. 変換先（おちびちゃんズ）が `Assets/夕時茶屋` 配下に入っているか確認
2. それでも出ない場合は **手動指定**で進める

### キャッシュを消して再生成する

自動候補は高速化のため、次のキャッシュを使います。

- `Library/Aramaa/OchibiChansConverterTool/FaceMeshCache.v7.json`

手順:

1. Unity を閉じる
2. 上記ファイルを削除
3. Unity を開き直して再度実行

> キャッシュは `Library` 配下に保存されるため、Git 管理対象ではありません（ローカル専用）。
{: .note }

---

## 髪・小物がズレる（MA Bone Proxy系）

- オプション「**MA Bone Proxyで設定している髪・小物がずれる場合に合わせる**」を **ON** にして再実行してください。

> ONにすると、複製後に MA Bone Proxy の処理を行い、ずれを軽減します（ツール内ヘルプ）。
{: .tip }

---

## 変換後、アップロードすると別アバターになる

仕様です。

- 変換対象は **複製物**（Ctrl+D相当）です
- 複製後、複製物の **Blueprint ID をクリア**します（元IDを引き継ぎません）

---

## ログを取りたい / Discordで相談したい

「ログを表示する」を ON にして実行すると、ログウィンドウが開きます。

- 「ログをクリップボードにコピー」ボタンで、そのまま貼り付けできます

相談時にあると助かる情報:

- Unity バージョン
- VRC SDK バージョン
- ツールバージョン
- 変換元アバター / 変換先おちびちゃんズ
- Console の赤エラー
- ツールログ
